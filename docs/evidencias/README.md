# Evidencias históricas

Este directorio conserva resultados de ejecuciones académicas anteriores. Son artefactos de evidencia, **no representan el estado actual de `main`**.

Reglas:

- no usar porcentajes o fallos históricos como estado vigente del producto;
- no añadir bases de datos, tokens, contraseñas ni datos personales;
- cada nueva evidencia debe indicar fecha, commit y entorno;
- los resultados actuales se obtienen ejecutando pytest y GitHub Actions;
- los archivos antiguos se mantienen por trazabilidad de tesis, no como tests activos.

Estado de calidad vigente: consulta el workflow `Android CI`, el README raíz y el último reporte de cobertura generado localmente.
