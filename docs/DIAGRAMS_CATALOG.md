# Catálogo de diagramas

La IA genera el inventario real con el comando `diagramas` y la API `/api/dev/project/diagrams`.

## Trabajo 1 — Núcleo del sistema

- arquitectura y despliegue;
- modelo entidad–relación;
- flujo de venta;
- diagrama de clases;
- mapa de endpoints.

## Trabajo 2 — IA y documentación

- arquitectura IA;
- herramientas por rol;
- flujo de documentos offline.

## Trabajo 3 — Calidad y ciclo de vida

- CI/CD;
- pirámide de testing;
- comparativa;
- evolución de cobertura;
- estados de sesión.

## Trabajo 4 — Telecomunicaciones

- puntos de medición Telecom;
- relación endpoint–DNS–TCP–TLS–HTTP–SQLite;
- continuidad local ante fallo WAN.

## Formatos

- `.puml`: fuente PlantUML editable;
- `.svg`: vector para tesis y presentaciones;
- `.png`: imagen para documentos convencionales.

## Uso en defensa

Cada figura debe indicar:

1. número y título;
2. fuente propia;
3. versión/commit;
4. componentes;
5. dirección del flujo;
6. relación con la hipótesis;
7. limitaciones.

No deben presentarse diagramas históricos como arquitectura actual sin verificar el commit asociado.
